택배에 있어서 1. 상품을 박스에 넣고 포장하면 2. 배송 정책을 정해야함
3. 최종 주소를 기입(영구적 보존) 4. 단지에 대한 경로 입력 5. 택배 운송
4번과 5번은 다른 위치에서 계속 바뀔 수 있음(다음 목적지에 대한 위치이므로)

네트워크에도 이러한 단계가 있음
5어플리케이션 단계 -> 유저 인터페이스 정의 (HTTP, FTP, DNS)
4트랜스포트 단계 -> 전송을 어떻게 확인, 오류를 어떻게 확인 (TCP, UDP)
3네트워크 단계 -> 네트워크 간 경로 설정 IPv4, IPv6 (관련 장치 : 라우터) 내부 아파트 단지
2데이터링크 단계 -> 네트워크 내 경로 설정 이더넷, PPP (관련 장치 : 스위치) 외부 아파트 단지
1피지컬 단계 -> 신호 처리 케이블 허브

이 5가지 단계가 TCP/IP 모델

3계층 주소 (IP) 3계층 장치 (라우터)
2계층 주소 (MAC) 2계층 장치 (스위치)
7계층 주소 (도메인 -> 인터넷 주소)

실제 패킷도 위와 같은 방식으로 만들어짐
단계를 지나면서 패킷이 만들어지고 도착 단계에서 하나씩 까면서 원본을 보게 됨
