lock을 이용해서 동시 다발적으로 이용해야하는 자원의 경우 자물쇠를 두고 한 사람만 이용할 수 있도록 하는 방법을 배움
하지만 실제 상황에서 그렇게 간단하지 않은게 여러 작업들은 각자의 연관성을 갖고 있기 때문에(ex) 주방에서 나온 요리를 손님 테이블에서 서빙하고 카드를 받아서 결제 프론트에서 결제) 쉽지 않음
그렇기 때문에 그런 작업들에 대해서 모두 lock을 잡으면 되지 않을까? 라는 생각을 할 수 있음
만약 그러지 않으면 게임 로직에서 DB의 내용을 가져올 때 쓰레드끼리 섞여서 문제가 생길 수 있음
이 경우 한쪽으로 몰리는 문제를 해결할 수 없음
모든 유저들이 분산되서 움직이면 좋겠지만 mmo의 경우 여러 유저가 모여서 같은 공간에서 패킷을 쏘게 되면 한 곳에 여러 쓰레드가 모이게 되고 lock에 의해 여러 쓰레드가 한번에 할 수 없음
즉 멀티쓰레드의 이점을 받을 수 없다는 점임
그래서 TLS(Thread Local Storage)의 개념이 필요함

한 존에 여러 사람이 모여서 같은 로직을 반복해서 전달하면 비효율적이 됨
이 때 우리는 같은 로직을 모아서 한번에 분배하는게 낫다는 생각을 할 수 있음

쓰레드 별로 스택 영역만 따로 사용하는 부분이 있고 힙영역(new)과 데이터영역(static)은 전역 메모리가 있음 (모든 쓰레드가 공유)
각자 사용하는 전역 변수가 있으면 TLS에 담아서 한번에 전달할 수 있음
